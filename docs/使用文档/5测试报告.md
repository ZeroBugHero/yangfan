# 测试报告

根据HttpRunner v4的测试报告，我们在扬帆测试平台上进行了专门的定制化处理，以更准确地反映测试的各个方面。在测试报告列表中，您可以看到运行节点、运行环境、用例类型以及执行类型等各项关键信息。

在测试报告详情部分，我们巧妙地使用不同的样式来区分执行总数、成功和失败的测试用例。而且，我们利用环形图表更直观地展示了测试用例的执行情况，让您一目了然。

相比其他测试平台，扬帆测试平台率先引入了时间分析功能。在测试报告详情中，我们增加了DNS解析、TCP连接、TLS握手、服务器端处理以及数据传输等各个阶段的时间信息，使您能方便地分析和定位到接口执行时间过长的问题。

此外，我们在测试报告详情中对任务、用例、步骤、接口进行了分层显示，这样可以让您更轻松地查看和理解整体用例的执行情况。

# 主要功能
扬帆测试平台的测试报告功能是我们团队专门针对实际测试需求进行深度定制的核心功能。针对各种类型的测试用例，无论是API、套件、单一用例、定时任务，还是性能测试，我们都能在测试报告中清晰展示。

测试报告支持不同的执行类型，包括保存调试、调试运行、后台运行以及定时运行。这种灵活性的设计，旨在帮助您以最适合您的方式进行测试。

扬帆测试平台的测试报告有两大主要功能：

1. 统一的测试报告：除性能测试数据外，所有其他类型的运行测试报告都将包含在内，为您提供全面、统一的测试反馈。
2. 测试报告详情：在详情部分，我们重点统计了“用例状态”和“接口状态”，让您快速了解每个测试用例和接口的运行情况。目前，我们还没有提供套件状态的统计，但我们正在积极探索和开发更多的功能，以帮助您更好地理解和控制测试流程。

我们的目标是使扬帆测试平台成为您最信赖的测试伙伴，无论您面对的是哪种测试需求，我们都期待为您提供最有效、最直观的测试报告
